Starting Zeb

Documentation on obtaining the Zeb software and installation can be found in the Zeb release notes, installation, and tutorial option off the Zeb home page.

Once Zeb is loaded onto your system, type startzeb in a free window. Note: Do not run startzeb in background.

The startzeb command starts up zeb through the following series of steps:

  • The first step is to try to find the project directory. If startzeb is given a parameter on the command line, it is expected to be the project directory. If not, startzeb first checks to see if the current directory is a project directory. Failing that, the user will be prompted for the name of the directory to use.

  • Next startzeb needs to find the data directory. If the environment variable DATA_DIR is set then its value will be used; otherwise the user is prompted.

  • Finally, all of the zeb processes are started up with a properly configured environment. Some project configurations include special startup code which will also be run by startzeb. For example, the STORMFEST configuration has code to deal with the possible existence of a CDROM on the system.
startzeb has numerous other options controlled mostly through environment variables. These are mostly of use to zeb developers and are beyond the scope of this tutorial.

What happens when zeb starts up

Zeb is made up of several processes which must be running for things to work properly. What you will see initially is the startzeb command starting up all of these processes. You will see output like: 
 Project dir is .  182 colors tweaked Starting core
zeb processes:
        message daemon [1] 4731
        event logger [2] 4734
        timer [3] 4736
[4] 4739
Loading:
     ...plot descriptions...
     ...button maps...
     ...display configurations ...
     ...ISS active area code ...
     ...the icon bar...
ready!
DM>
The last line is the display manager prompt. You may type commands at that prompt.

A set of windows will come up on the workstation. The set of windows displayed is dependant on the "DEFAULT_CONFIG" environment variable. In general the Event Logger, IconBar , and a Graphics Window will be displayed.
