Starting Zebra

Documentation on obtaining the Zebra software and on installation can be found in the Zebra release notes and Installation manual, both of are accessible through the Zebra home page.

Once Zebra is loaded onto your system, type zstart in a free window. Note: Do not run zstart in background.

The zstart command starts up Zebra through the following series of steps:

  • The first step is to try to find the project directory. If zstart is given a parameter on the command line, it is expected to be the project directory. If not, zstart first checks to see if the current directory is a project directory. Failing that, the user will be prompted for the name of the directory to use.

  • Next zstart needs to find the data directory. If the environment variable DATA_DIR is set then its value will be used; otherwise the user is prompted.

  • Finally, all of the Zebra processes are started up with a properly configured environment. Some project configurations include special startup code which will also be run by zstart. For example, the STORMFEST configuration has code to deal with the possible existence of a CDROM on the system.
zstart has numerous other options controlled mostly through environment variables. These are useful mainly to Zebra developers, but an overview of each of them can be found in the Introducing Zebra manual.

What happens when Zebra starts up

Zebra is made up of several processes which must be running for things to work properly. What you will see initially is the zstart command starting up all of these processes. You will see output like: 

Project dir is . 
182 colors tweaked
Starting core Zebra processes:
        message daemon [1] 4731
        event logger [2] 4734
        timer [3] 4736
[4] 4739
Loading:
     ...plot descriptions...
     ...button maps...
     ...display configurations ...
     ...ISS active area code ...
     ...the icon bar...
ready!
DM>
The last line is the display manager prompt. You may type commands at that prompt.

A set of windows will come up on the workstation. The set of windows displayed is dependant on the "DEFAULT_CONFIG" environment variable. In general the Event Logger, IconBar , and a Graphics Window will be displayed.
