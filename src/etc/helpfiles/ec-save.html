Saving a display configuration

Once you get the display to a state you like, you should save the configuration so you don't have to do all that work again.

If your icon bar provides the configuration editing widget:

Then click on Save configuration now. If your iconbar has a Save display window entry, select that now. In either case the following widget will appear:
The name for the configuration appears on the top line, and will normally be the name of the configuration you started with. If this is a brand new configuration, you almost certainly want to change this name to avoid a conflict with the predefined configuration. If you are modifying an existing configuration, you may want to leave the name as-is.

The second line contains the directory into which your configuration will be saved. The default value is the directory .zeb_configs in your home directory (which will be created, if necessary). If you save configurations into that directory, you will be able to recall them, but other users will not see them (unless they take additional steps). You may edit this line to save the configuration into the project configuration directory if you have access there; it is usually called dconfig in the project directory.

If you save a configuration in your .zeb_configs directory with the same name as a configuration in the project directory, the new one will take precedence for you, and the predefined configuration will no longer be accessible. For this reason, it is best to pick new, unique names for your own configurations.

When everything is set, click on Do it to save the configuration.
